# SyncTerm‑IDE v4.0.0 — Release Notes

## Highlights

* **Web 版本格対応** — ブラウザで動作する Web フロント（syncterm-web）と FastAPI バックエンド（backend）。デスクトップ版と並行利用可能。
* **リバーストンネル（RT）モード** — Watcher が Relay へ逆方向トンネルを張り、コマンド・ログ・ファイル取得を HTTP で即時送受信。低遅延・rsync 待ち不要。
* **トンネル付き一括起動** — `./scripts/start-web-with-tunnel.sh` および `npm run dev:tunnel` で SSH トンネルと Vite を同時起動。終了時にトンネルも自動停止。
* **Staged キャッシュ削除** — Web の SessionBar から「キャッシュ削除」で Relay / Watcher の staged ファイルを一括削除。
* **画像プレビュー・画像タブ** — 画像を Blob 取得でプレビュー。画像用タブを Editor と分離して表示。

## Added

* **Web 版**
  * React フロント（syncterm-web）と FastAPI バックエンド（backend）。ブラウザから Relay 経由で Watcher を操作可能。
* **RT モード**
  * `watcher_manager_rt.sh` と `command_watcher_rt.py`。Watcher が Relay へ SSH リバーストンネルを張り、コマンド・ログを HTTP で即時送受信。
* **トンネル付き一括起動**
  * `.env.tunnel` で `TUNNEL_SSH` を設定し、`./scripts/start-web-with-tunnel.sh` または `npm run dev:tunnel` でトンネル + Vite を同時起動。
* **キャッシュ削除**
  * SessionBar の「キャッシュ削除」で Relay 上の staged および Watcher 側の staged を一括削除。
* **画像プレビュー**
  * 対応画像を Blob で取得しプレビュー表示。画像タブを Editor と分離。
* **設定例**
  * `config.ini.example`、`.env.tunnel.example` を追加。個人情報を含まない。

## Changed

* **README** — Web 版を推奨とする記述に変更。Quick Start で Web 版を先に案内。Features・注意事項を現行仕様に合わせて整理。
* **デプロイ** — `scripts/deploy_backend.sh` のデフォルト REMOTE_DIR を `~/SyncTerm-IDE` に変更。

## Fixed

* RT モードで Relay にセッション dir が無い場合にキャッシュ削除が 404 で失敗する問題を修正。Relay 側にセッションが無くても Watcher 側のみ削除可能に。

## Notes

* Web 版ターミナルの Local モードはコマンド実行なし（表示のみ）。Remote モードは Watcher 上で実行される。
* 詳細なセットアップは [WEB-SETUP.md](../WEB-SETUP.md) を参照。
