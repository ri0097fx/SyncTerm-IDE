# SyncTerm‑IDE v3.0.0 — Release Notes

## Highlights

* **Docker 統合** — GUI から Host / Docker Run / Docker Exec を切り替える実行環境機能を追加。イメージ取得やマウント設定を GUI 上で完結できる。
* **ログの逐次同期** — ログ読み取りをスレッド化し、実行中プロセスの出力をよりリアルタイムに取得可能。
* **Python 実行ボタン** - エディタ上で開いているスクリプトをそのままワンクリックで実行。
* **シンボリックリンク対応の強化:** - リンク先フォルダ配下でのファイル作成や削除が正しく GUI 側に反映されるように改善。
* **大規模リファクタリング** - `gui_terminal.py` 主体の構造から、`main.py` + `components/` + `sync_services/` のモジュール型構成へ移行。保守性と拡張性が大幅に向上。

## Added

* **Docker 実行環境サポート**
  * Host / docker_run / docker_exec の 3 モードを GUI から動的に切り替え可能。
  * Docker イメージ一覧・マウント設定・コンテナ状態を GUI 内で完結。
* **ログ逐次同期**
  * 別スレッドでログを取得し、UI のフリーズを防ぎつつリアルタイム性能を強化。
* **Python 実行ボタン**
  * "▶ Run" ボタンを追加し、現在開いている Python ファイルを即実行可能。
* **リンクフォルダの完全対応**
  * シンボリックリンク配下での新規作成 / 削除を正しく同期。

## Changed

* **アプリ構造の全面刷新**

  * エントリポイントを `main.py` に刷新。
  * GUI 構成要素は `components/` に分割、同期ロジックは `sync_services/` に分離。
  * エディタ・ターミナル・ファイルツリーの構造を統合的に整理。
  
* **Conda 表示修正**
  * Anaconda が存在しない環境で "(base)" が表示されてしまう問題を改善。

## Fixed

* シンボリックリンクの扱いがフォルダとして認識されない問題を修正。
* ログ読み取りによる UI フリーズを大幅に軽減（非同期化）。

## Breaking Changes
* 起動スクリプトが `gui_terminal.py` → `main.py` へ変更。
* GUI 内部 API が刷新されたため、2.x 系の内部向けフックは一部非互換。

## Notes

* 既存の 2.x セッションはそのまま使用可能だが、Docker 関連設定は新規に作成する必要あり。
* 旧 GUI のショートカットを利用している場合は main.py を指すように更新すること。
